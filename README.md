### Задание 1

`CPU Usage (процент загрузки, load average)
Вычисления сильно нагружают CPU, нужно отслеживать приближение к 100% и деградацию производительности.
HTTP Requests (RPS, latency p50/p95/p99, error rate 4xx+5xx)
Основной протокол взаимодействия. Метрики показывают доступность сервиса, скорость ответов и количество ошибок.
Disk Usage (свободное место на диске)
Отчёты сохраняются на диск, что может привести к заполнению и падению сервиса.
Memory Usage (RSS/Heap + OOM)
При CPU-интенсивных задачах память часто становится следующим bottleneck.
Количество активных вычислений / очередь задач (если есть)
Позволяет видеть реальную нагрузку на бизнес-логику и вовремя выявлять заторы.`
